# Documentación funcional y técnica para MUSES Web

Este directorio concentra artefactos de trabajo y documentación detallada que sirvieron de base para la construcción de la aplicación `muses-web`.

**Nota Importante:** La arquitectura del frontend ha sido consolidada en un único documento maestro. Para una visión completa y actualizada, por favor consulta: **[Arquitectura Detallada del Frontend](../../docs/arquitectura_detallada_frontend.md)**.

## Referencias disponibles (Histórico)

- `bd/layout_V01.md`: Definición completa de campos, máscaras y catálogos.
- `inscripcion_diccionario_campos.md`: Resumen de campos clave para formularios.
- `inscripcion_reglas_validacion.md`: Lista de validaciones para captura.
- `inscripcion_catalogos_referenciados.md`: Inventario de catálogos oficiales.
- `cu_formulario_inscripcion_registro.md`: Caso de uso para el formulario manual.
- `muses_web_diseno_componentes.md`: Catálogo de componentes y servicios.
- `muses_web_plan_trabajo.md`: Plan de trabajo Scrum inicial.

## Cómo mantener esta documentación
1.  **Priorizar el Documento Maestro:** Cualquier decisión de arquitectura relevante (nuevos módulos, integraciones, librerías) debe reflejarse en el documento de **[Arquitectura Detallada del Frontend](../../docs/arquitectura_detallada_frontend.md)**.
2.  **Actualizar Artefactos Específicos:** Los documentos en este directorio pueden ser actualizados si cambian los layouts o catálogos específicos que describen.
3.  **Mantener Trazabilidad:** Mantén enlaces cruzados entre historias de usuario, pruebas y estos documentos para facilitar la trazabilidad normativa y técnica.
